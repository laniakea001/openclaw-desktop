## 架构概述

```
┌─────────────────────────────────────────────────────────┐
│                    用户界面 (飞书/Telegram/Discord)      │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│                     OpenClaw Gateway                    │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────┐ │
│  │  消息路由   │  │  Skill 引擎 │  │  模型网关       │ │
│  └─────────────┘  └─────────────┘  └─────────────────┘ │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│                      AI 模型 (MiniMax/DeepSeek/Claude)  │
└─────────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 职责 |
|------|------|
| Gateway | 消息路由、会话管理 |
| Skill Engine | 技能加载、执行 |
| Model Gateway | 模型调用、响应处理 |
| Channel Adapter | 渠道消息适配 |

## 核心概念

### 1. Skill（技能）

Skill 是 OpenClaw 的扩展单元，每个 Skill 负责特定功能：

```typescript
// Skill 结构
{
  name: "skill-name",
  description: "技能描述",
  triggers: ["触发关键词", "/command"],
  actions: [
    { type: "tool", name: "tool_name", params: {} }
  ]
}
```

### 2. Channel（渠道）

支持的渠道：
- 飞书 (Feishu)
- Telegram
- Discord
- Slack
- WhatsApp
- Signal

### 3. Model（模型）

支持的模型：
- MiniMax M2.5
- DeepSeek
- Claude (Anthropic)
- OpenAI

## Skill 开发

### 创建新 Skill

```bash
# 使用 skill-creator
clawhub skill create my-skill
```

### Skill 文件结构

```
my-skill/
├── SKILL.md           # Skill 定义
├── index.ts           # 入口文件
├── actions/           # 动作定义
│   ├── action1.ts
│   └── action2.ts
└── tests/             # 测试
    └── index.test.ts
```

### 示例：简单 Skill

```typescript
// my-skill/index.ts
export const skill = {
  name: "hello",
  description: "打招呼技能",
  triggers: ["你好", "/hello"],
  
  async handle(context) {
    const { message, reply } = context;
    await reply(`你好！我是 ${context.config.name}`);
  }
};
```

### Skill 触发条件

| 类型 | 示例 | 说明 |
|------|------|------|
| 关键词 | "天气" | 消息包含关键词 |
| 正则 | /^\\/.*$/ | 正则匹配 |
| 意图 | "查天气" | AI 意图识别 |


## 最佳实践

### 1. Skill 设计原则

- ✅ 单一职责：一个 Skill 只做一件事
- ✅ 低耦合：Skill 之间通过事件通信
- ✅ 可测试：编写单元测试

### 2. 错误处理

```typescript
try {
  await skill.execute(context);
} catch (error) {
  logger.error("Skill execution failed", { skill: skill.name, error });
  await context.reply("抱歉，出错了，请稍后重试");
}
```

### 3. 速率限制

```typescript
// 在 Skill 中添加速率限制
const rateLimit = new RateLimiter({
  windowMs: 60 * 1000,  // 1 分钟
  max: 10               // 最多 10 次
});

if (!rateLimit.check(userId)) {
  await context.reply("请求过于频繁，请稍后再试");
  return;
}
```

### 4. 日志记录

```typescript
const logger = {
  info: (msg, data) => console.log(`[INFO] ${msg}`, data),
  error: (msg, data) => console.error(`[ERROR] ${msg}`, data),
  debug: (msg, data) => console.debug(`[DEBUG] ${msg}`, data),
};
```

## 常见问题

### Q1: 如何调试 Skill？

使用 `clawhub skill dev` 启动开发模式，支持热重载。

### Q2: 如何添加新的渠道？

参考 `channels/feishu` 的实现，实现 `ChannelAdapter` 接口。

### Q3: 模型调用失败怎么办？

检查：
1. API Key 是否正确
2. 账户余额是否充足
3. 网络是否正常
